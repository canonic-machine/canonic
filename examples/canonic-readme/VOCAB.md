## Document Governance Terms

### engaging documentation
Documentation that maintains reader interest through clear structure, concrete examples, and progressive complexity.

### inheritance demonstration
Example showing how downstream CANONs acquire and specialize constraints from upstream CANONs.

### nested triad
Triad structure within parent triad structure, demonstrating recursive governance pattern. Child inherits from parent, may specialize defaults.

### reference resolution
Property that all file references, links, and citations point to existing targets. Prevents broken links.

### runnable code example
Code snippet that can be executed without modification. Must be tested and functional.

### technical accuracy
Documentation content that correctly describes system behavior, constraints, and implementation details.
